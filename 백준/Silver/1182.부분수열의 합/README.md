### 문제 분류
알고리즘 - 재귀 호출 (Recursive Call), 브루트포스 탐색

 ### 문제 설명
 N개의 정수로 이루어진 수열이 있을 때, 크기가 양수인 부분수열 중에서 그 수열의 원소를 다 더한 값이 S가 되는 경우의 수를 구하는 프로그램을 작성하시오. 
 
 ### 입력
 첫째 줄에 정수의 개수 N과 정수 S가 주어진다. ($1 \le N \le 20$, $|S| \le 1,000,000$) 둘째 줄에 N개의 정수가 빈칸을 사이에 두고 주어진다.\

  ### 출력
 첫째 줄에 합이 S가 되는 부분수열의 개수를 출력한다.

### 코드

```
N, S = map(int, input().split())
nums = list(map(int, input().split()))

def count_subsets(idx, current_sum):   
    if idx == N:
        return 0

    count = 0

    if current_sum + nums[idx] == S:
        count += 1
        
    # 현재 숫자를 더하는 경우
    count += count_subsets(idx + 1, current_sum + nums[idx])

    #현재 숫자는 그냥 건너뛰는 경우
    count += count_subsets(idx + 1, current_sum)

    return count


print(count_subsets(0, 0))
```
 ### 풀이
 이 문제는 재귀호출을 활용하여 수열 내 모든 부분집합을 탐색해야합니다.
 수열 내 각 원소에 대하여 선택O/선택X 두 선택지를 가지는 이진트리 형태로 로직을 구현하였습니다.
 아무것도 선택되지않는(합이0인경우)경우가 카운트되는 오류를 방지하기위해 current_sum + nums[idx] == S를 각단계에서 체크되도록하였습니다.
 재귀의 종료 조건으로 idx == N을 설정하여 리스트의 끝에 도달했을 때 안전하게 리턴되도록 구현하였습니다

 ### 배운점
 각 원소에 대해 '선택'과 '비선택'이라는 두 선택지를 가지는 재귀 호출이 조가 전체적으론 이진 트리형태를 띤단걸 구현해보며 경험하였습니다.
 함수가 호출될때마다 파라미터가 스택에 쌓이고 리프노드까지 내려갔다가 다시 올라오는 DFS과정을 지켜보며 재귀의 실행 흐름을 다시금 정리할수있었습니다.
