### 문제 설명

<p>N개의 정수 A[1], A[2], …, A[N]이 주어져 있을 때, 이 안에 X라는 정수가 존재하는지 알아내는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 자연수 N(1 ≤ N ≤ 100,000)이 주어진다. 다음 줄에는 N개의 정수 A[1], A[2], …, A[N]이 주어진다. 다음 줄에는 M(1 ≤ M ≤ 100,000)이 주어진다. 다음 줄에는 M개의 수들이 주어지는데, 이 수들이 A안에 존재하는지 알아내면 된다. 모든 정수의 범위는 -2<sup>31</sup> 보다 크거나 같고 2<sup>31</sup>보다 작다.</p>

### 출력 

 <p>M개의 줄에 답을 출력한다. 존재하면 1을, 존재하지 않으면 0을 출력한다.</p>

###  풀이
초기에는 코드를 다음과 같이 작성했습니다.

```python
N = int(input())
A = list(map(int, input().split())) # 리스트로 입력 받음
M = int(input())
B = list(map(int, input().split()))

for i in range(M):
    if B[i] in A: # 리스트 전체를 하나하나 탐색 (O(N))
        print(1)
    else:
        print(0)
```

다만 이렇게 작성하면 '시간초과'문제가 발생했습니다. 

리스트에서 in연산자를 사용할경우 O(N)시간까지 걸릴수있기에 M번반복시 O(MxN)이 되기에 이 문제가 발생하는것이였습니다.

set()를 사용하면 해시테이블구조를 이용하기에 시간이 O(1)로 줄어들수있으므로 O(MxN)->O(M)까지 줄어들수있기에 

이후 A = list(map(int, input().split())) 를 -> A = set(map(int, input().split()))로 수정하였습니다.

### 배운점
백준 문제를 처음 풀어보면서, 단순히 정답 로직을 세우는 것보다 그 로직을 얼마나 효율적으로 구현하느냐가 백준에선 중요하단걸 느꼈습니다.처음에는 평소 익숙했던 리스트를 사용해 단순히 in 연산자로 탐색했으나, 데이터의 양이 많아지자 여지없이 '시간 초과'가 발생했습니다. 원인을 분석하는 과정에서 리스트(O(N))와 세트(Set)의 해시 탐색(O(1)) 사이의 속도 차이를 직접 체감할 수 있었습니다.학교 수업 시간에 이론으로만 배웠던 해시 테이블 개념이 파이썬의 set 자료형을 통해 어떻게 실전 알고리즘에 적용되어 성능을 최적화하는지 이번에 경험하였습니다. 앞으로 백준 문제를 풀 때 시간 복잡도와 적절한 자료구조 선택을 항상 고려해야한단걸 배웠습니다.
